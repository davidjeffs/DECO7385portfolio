OVERVIEW

See below for details on the overall concept and individual designs.

What is it?

Social Provocateurs have developed a set of installations that look to provoke a reflection on the value that we get from modern day communication technologies, and the infinite amount of information that they provide us. This has been achieved with the design of two exploratory concepts, The Social Dial and Twitter Tapper.

The Context

Both these concepts have been designed for the home context. It is imagined that a user would have them set up, such that if they wanted to use the phone they would access their individual Facebook account.

Further more they were designed as a contrast between current communication technologies, and archaic ones of the past. It was originally hoped that these would provoke different reflections based on the user generations. This was later analysed in the reflective report and shown to be invalid.

TWITTER TAPPER

The Twitter Tapper is a combination of the social media network Twitter, and the obsolete communication method of morse code. By applying the long work flow and interaction method of a morse code to the message posting of Twitter, the design provokes reflection on the value of each message and whether it is worth typing. The lack of error correction provides a focus on the public nature of our social network activities and that once a message is posted it is unable to be taken back (delete functions do not always work).

Design Decisions

Logo signifier

The use of the Twitter logo on the button was both a signifier and a mapping to Twitter.

Attention

The use of headphones was based on user testing, and is used to focus users attention on the task. This was based on feedback provided in the form of beeps.

Aesthetics

The context of the project is important. Brass and dark wood were used as cognitive support of the concept.

Morse Alphabet

The morse code alphabet was designed for high usability to ensure attention is placed on the slow work flot of the morse code machine.

Error Correction

Whilst traditional morse code has error correction, this was removed from Twitter Tapper to increase focus that once a message was sent it could not be taken back.

How it works

Technical

The morse code is basic button connected to an Arduino. The Arduino code analyses the code and produces text. A python framework is then used to post this to Twitter (code by Will).

Interaction

User uses twitter button for long and short presses that are mapped to individual letters as shown on the alphabet.

User Experience

Expected

Based on the research it was expected that users would feel frustrated at the long process required to input a message. However due to this long process, users would also pause to consider the value of the message being inputted.

Outcome

Based on the frustration, users would then have a reflection on the ease of communication, but also the the value of the message that they are sending. Is the message worth the long process?

THE SOCIAL DIAL

The Social Dial is a combination of the social media network Facebook, and the archaic communication method of a rotary phone. This allows users to interact with a Facebook account without the use of a screen. They are able to message friends, post and read status updates, read the news feed, and post and read comments. By implementing decision making at each step of the workflow the design provokes users to reflect on the value of each message that is posted and viewed. Users are forced to listen to each individual item on the news feed.

Design Decisions

Robotic Voice

The use of a robotic voice is the removal of emotion. This was an assumption that was confirmed via user testing. This places focus on the value of the message.

Workflow

The workflow of the telephone attempted to match the WIMP workflow of Facebook. This way the focus is placed on the interaction with the telephone and the information that it provides.

Voice Interaction

The user experience was designed to be frustrating. However askign the user to use the dial to input text via a t3 interaction was assumed to be too time consuming.

Threading

Threading was introduced into the code such that users are able to dial a number before the voice commands complete.

How it works

Technical

The telephone electronics are connected to an Arduino. This then connects to a computer running a python framework which takes input from the arduino and Google Cloud Text-to-speech.

Interaction

The user lifts the handset to begin the interaction. The voice prompts are then followed to interact with the endless amont of information that Facebook offers. Users are able to enter text through the Google speech-to-text API.

User Experience

Expected

Based on the research it was expected that users would feel frustrated at the long process required interact with the Facebook work flow (and information). However due to the nature of interaction, users are forced to listen to each message, giving greater value to the information contained.

Outcome

Based on the frustrating long process, users would then have a reflection on the ease of communication via Facebook, but also the the value of the message that they are sending and receiving.
